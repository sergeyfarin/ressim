## Scope

This contract freezes the store-facing schema for Option B (Unified Preset + Customize) so UI slices can build incrementally without changing core interfaces.

## Store-facing fields

`scenarioSelection`
- `basePreset`
  - `key: string`
  - `mode: 'dep' | 'wf' | 'sim' | 'benchmark'`
  - `source: 'facet' | 'benchmark' | 'custom'`
  - `label: string`
  - `toggles: ToggleState`
  - `benchmarkId: string | null`
- `benchmarkProvenance: BenchmarkProvenance | null`
- `setBenchmarkProvenance(provenance: BenchmarkProvenance | null)`

`parameterState`
- `parameterOverrides: Record<string, { base: unknown; current: unknown }>`
- `parameterOverrideGroups: Record<
  'grid' | 'initial' | 'fluids' | 'permeability' | 'relperm' | 'wells' | 'stability' | 'physics' | 'analytical',
  string[]
>`
- `parameterOverrideCount: number`

`runtimeState`
- `analyticalStatus`
  - `level: 'reference' | 'approximate' | 'off'`
  - `mode: 'waterflood' | 'depletion' | 'none'`
  - `warningSeverity: 'none' | 'notice' | 'warning' | 'critical'`
  - `reasonDetails: Array<{ code: string; message: string; severity: 'notice' | 'warning' | 'critical' }>`
  - `reasons: string[]`

## Contract intent

- `basePreset` identifies the current generated baseline from facets/benchmark/custom state.
- `parameterOverrides` is deterministic diff data against the baseline generated by current toggles.
- `benchmarkProvenance` is reserved for benchmark-to-custom clone lineage.
- `analyticalStatus` is permissive by design (`reference` or `approximate` with reasons), with `off` reserved for explicit disable flows.
- `warningSeverity` summarizes caveat intensity for UI policy (`none`, `notice`, `warning`, `critical`) while `reasonDetails` provides explicit machine-readable caveats.
